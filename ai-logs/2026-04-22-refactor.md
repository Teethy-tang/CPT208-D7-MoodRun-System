# AI Development Log - Front-End Refactor

Date: 2026-04-22

## Purpose

This log records the primary AI-assisted prompt used to reorganise the MoodRun prototype into a cleaner coursework-ready front-end structure.

## Primary Prompt Summary

The project started as a single HTML file containing all HTML, CSS, and JavaScript. The prompt asked Codex to:

- Inspect the existing prototype.
- Preserve the current visual design and interaction flow.
- Separate concerns into `index.html`, `css/`, `js/`, and `assets/`.
- Keep the app lightweight with plain HTML, CSS, and JavaScript.
- Modularise JavaScript into routing, mood input, run plan selection, running simulation, summary generation, wisdom book, profile/history, and storage helpers.
- Prepare the project for localStorage state handling, responsive design, deployment, README documentation, and AI logs.
- Avoid overbuilding new features during the refactor.

## Resulting Core Components

- `index.html`: screen markup for the single-page app.
- `css/styles.css`: extracted visual styling and responsive rules.
- `js/app.js`: main app coordinator.
- `js/data.js`: static app content and plan data.
- `js/effects.js`: visual and timer effects.
- `js/router.js`: screen switching.
- `js/runTracker.js`: running simulation and run formatting.
- `js/storage.js`: localStorage helper functions.

## Human Review Notes

The structure was then adjusted to better match the coursework System Development requirement by keeping one root `README.md`, removing the extra `pages/README.md`, and adding the required `ai-logs/` folder.
